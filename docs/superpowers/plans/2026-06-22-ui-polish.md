> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Polish visual consistency across the header, lessons page, availability page, and login page within the existing "editorial jazz" design system, with no new tokens/dependencies and no behavior changes.

**Architecture:** Pure presentational edits (JSX className changes + small derived-data additions for "today" highlighting) in four existing client components. No new files, no API/data model changes.

## Global Constraints

- No new colors, fonts, or npm dependencies — reuse existing CSS variables/utility classes from `src/app/globals.css`.
- No changes to data fetching, API routes, or component logic/behavior (filters, CRUD, auth flow must work identically after the change).
- This repo has no automated test suite (`package.json` defines no `test` script and ships no test framework) — verification is manual: run `npm run dev` and visually inspect each page at mobile (~390px) and desktop (~1280px) widths.
- Run `npm run build` after all tasks as a type/lint safety net (the project uses TypeScript + ESLint via `next build --webpack`).
